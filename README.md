# 🔐 ТГ-бот менеджер паролей

Telegram-бот для безопасного хранения паролей, написанный на **C++17**. Пароли
шифруются криптостойким алгоритмом **AES-256-GCM** (библиотека OpenSSL) на ключе,
который выводится из загружаемого пользователем файла-ключа. Зашифрованные данные
хранятся в локальной базе **SQLite**.

## Назначение

Программа представляет собой Telegram-бота, который позволяет пользователю
хранить пароли от различных ресурсов (сайтов, сервисов) в зашифрованном виде.
Ключ шифрования не хранится в программе: пользователь предоставляет файл-ключ при
каждом начале работы, и без него расшифровать пароли невозможно.

## Возможности

| Команда | Назначение |
|---------|------------|
| `/start` | Приветствие и запрос файла-ключа |
| *(отправка файла)* | Загрузка файла-ключа, начало сессии |
| `/add <ресурс> <пароль>` | Создать новый ресурс с паролем |
| `/get <ресурс>` | Получить пароль ресурса (с кнопкой «Копировать») |
| `/change <ресурс> <новый_пароль>` | Изменить пароль ресурса |
| `/delete <ресурс>` | Удалить ресурс |
| `/list` | Показать все ресурсы с паролями |
| `/help` | Справка по командам |

Дополнительно:

- 📋 У каждого пароля есть inline-кнопка **«Копировать»** — пароль показывается во
  всплывающем окне, откуда его удобно скопировать.
- ⏲️ **Тайм-аут сессии**: ключ автоматически удаляется из памяти после периода
  бездействия (по умолчанию 30 минут).
- 🗑️ Сообщения с паролями, отправленные пользователем, удаляются ботом.

## Как это работает

```
Файл-ключ ──PBKDF2-HMAC-SHA256──► ключ 256 бит ──┐
                                                  │
Пароль ──AES-256-GCM(ключ, случайный IV)──► base64(IV ‖ ciphertext ‖ tag) ──► SQLite
```

1. Пользователь загружает **файл-ключ**. Из его содержимого алгоритмом
   **PBKDF2-HMAC-SHA256** (600 000 итераций) выводится 256-битный ключ.
2. При сохранении пароля генерируется случайный вектор инициализации (**IV**),
   пароль шифруется **AES-256-GCM**, результат кодируется в Base64 и сохраняется.
3. При чтении пароль расшифровывается и проверяется **тег аутентификации GCM**:
   при неверном ключе или повреждённых данных операция завершается ошибкой, а не
   возвращает мусор.

## Архитектура

```
src/
├── main.cpp              # Точка входа, разбор аргументов командной строки
├── common/
│   └── Exceptions.h      # Иерархия исключений приложения
├── crypto/
│   ├── CryptoEngine.h    # PBKDF2 + AES-256-GCM (OpenSSL)
│   └── CryptoEngine.cpp
├── storage/
│   ├── Database.h        # Обёртка над SQLite (CRUD, prepared statements)
│   └── Database.cpp
├── session/
│   ├── SessionManager.h  # Хранение ключей в памяти + тайм-аут
│   └── SessionManager.cpp
└── bot/
    ├── BotHandler.h      # Обработчики команд и callback-ов Telegram
    └── BotHandler.cpp
tests/                    # Модульные тесты (doctest)
docs/                     # Doxyfile + сгенерированная HTML-документация
```

Модули `crypto`, `storage`, `session` собираются в статическую библиотеку
`aaip_core`, которую используют **и** исполняемый файл бота, **и** тесты — то есть
тестируется ровно тот код, из которого собирается основной проект.

## Зависимости

| Библиотека | Назначение | Как подключается |
|------------|------------|------------------|
| [OpenSSL 3](https://www.openssl.org/) | Шифрование AES-256-GCM, PBKDF2 | системная (`find_package`) |
| [Boost](https://www.boost.org/) | Требование `tgbot-cpp` | системная (`find_package`) |
| [PostgreSQL 14+](https://www.postgresql.org/) | СУБД (сервер + libpq) | системная (`find_package`) |
| [tgbot-cpp](https://github.com/reo7sp/tgbot-cpp) | Telegram Bot API | **CMake FetchContent** |
| [libpqxx 7.9](https://github.com/jtv/libpqxx) | C++ обёртка над libpq | **CMake FetchContent** |
| [doctest](https://github.com/doctest/doctest) | Модульные тесты | **CMake FetchContent** |

tgbot-cpp, libpqxx и doctest автоматически загружаются и собираются средствами
CMake — ручная установка не требуется. PostgreSQL устанавливается системно.

## Установка и сборка

### Предварительные требования (macOS / Homebrew)

```bash
brew install cmake openssl@3 boost postgresql@14
```

После установки PostgreSQL создайте базы данных:

```bash
brew services start postgresql@14
createdb aaip        # рабочая БД
createdb aaip_test   # БД для тестов
```

> Для генерации документации дополнительно: `brew install doxygen graphviz`.

### Сборка

```bash
cmake -S . -B build
cmake --build build -j
```

После сборки появятся:

- `build/aaip_bot` — исполняемый файл бота;
- `build/aaip_tests` — набор модульных тестов.

## Запуск

1. Создайте бота через [@BotFather](https://t.me/BotFather) командой `/newbot` и
   получите токен.
2. Запустите бота, передав токен:

```bash
./build/aaip_bot --token "123456:ABC-DEF..."
```

или через переменную окружения:

```bash
export TELEGRAM_BOT_TOKEN="123456:ABC-DEF..."
./build/aaip_bot
```

### Параметры командной строки

Пользователь может задать строку подключения и прочие параметры при запуске:

```
--token <токен>      Токен Telegram-бота (или переменная TELEGRAM_BOT_TOKEN)
--db <url>           Строка подключения к PostgreSQL
                     (по умолчанию: postgresql://localhost/aaip)
--timeout <секунды>  Тайм-аут сессии в секундах (по умолчанию: 1800)
--help               Показать справку
```

Пример с нестандартной БД и паролем:

```bash
./build/aaip_bot --token "..." \
    --db "postgresql://myuser:mypass@localhost:5432/vault" \
    --timeout 600
```

## Сценарии использования

Типичный диалог с ботом в Telegram:

```
Пользователь: /start
Бот:          🔐 Менеджер паролей. Отправьте файл-ключ для начала работы…

Пользователь: [отправляет файл key.txt]
Бот:          ✅ Ключ принят. Теперь вы можете управлять паролями.

Пользователь: /add github MyS3cretPass!
Бот:          ✅ Пароль для github сохранён.
              (исходное сообщение с паролем удаляется)

Пользователь: /get github
Бот:          🔑 github
              MyS3cretPass!
              [ 📋 Копировать ]

Пользователь: /list
Бот:          🔐 Ваши ресурсы:
              ▫️ github: MyS3cretPass!
              [ 📋 github ]

Пользователь: /change github NewPass456
Бот:          ✅ Пароль для github обновлён.

Пользователь: /delete github
Бот:          ✅ Ресурс github удалён.
```

Если пользователь вернётся даже через месяц — пароли никуда не денутся. Нужно
лишь снова отправить **тот же** файл-ключ, и все записи расшифруются.

## Тесты

Тесты написаны с использованием модуля **doctest** и запускаются стандартной
командой `ctest` из каталога сборки:

```bash
cd build
ctest --output-on-failure
```

Тесты базы данных подключаются к `postgresql://localhost/aaip_test` (можно
переопределить через переменную окружения `TEST_DATABASE_URL`):

```bash
TEST_DATABASE_URL="postgresql://user:pass@host/mytest" ctest --output-on-failure
```

Если PostgreSQL недоступен — тесты БД автоматически пропускаются, остальные
(crypto, session) выполняются в штатном режиме. Покрыты положительные и
отрицательные сценарии для модулей `CryptoEngine`, `Database` и `SessionManager`
(25 тест-кейсов).

## Безопасность

- Ключ шифрования **никогда не сохраняется на диск** — только в оперативной памяти.
- **AES-256-GCM** обеспечивает не только конфиденциальность, но и аутентификацию
  данных (защита от подмены).
- Для каждого пароля используется **случайный IV** — одинаковые пароли дают разный
  шифротекст.
- **PBKDF2 с 600 000 итераций** затрудняет перебор по файлу-ключу.
- Все SQL-запросы выполняются через **параметризованные запросы** libpqxx
  (`exec_params`) — защита от SQL-инъекций.
- Токен бота берётся из аргумента или переменной окружения и **не хранится в коде**.
